# Post-Merge Checklist

After any PR merge, run this checklist in order.

- [ ] Update `kai-pm/STATUS.md` (current phase, in-flight PRs, next actions, timestamp)
- [ ] Update `kai-pm/SEQUENCE.md` step status if unlock/progress changed
- [ ] Append any new policy call to `kai-pm/DECISIONS.md` (never edit old entries)
- [ ] Update `CHANGELOG.md` Unreleased section
- [ ] Update `README.md` only if public-facing behavior or project-state summary changed
- [ ] Refresh `kai-pm/METRICS.md` values that are known
- [ ] Re-check `kai-pm/RISKS.md` statuses and mitigations
- [ ] Confirm automation/workflows remain aligned with PM rules
